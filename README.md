# TeamDJ

수학 학원 학습 관리 시스템(LMS). 랜딩 페이지 + 스태프 관리 웹 + 학생/학부모 대시보드 + 모바일 앱이 하나의 Supabase 데이터베이스를 공유하는 모노레포.

## 구조

```
apps/
  web/   # Next.js 16 — 랜딩, 스태프 관리자, 학생 대시보드
  app/   # Expo (React Native) — 학생/학부모 모바일 앱
supabase/
  migrations/   # DB 스키마 (append-only, migration-guard CI로 강제)
```

| 스택 | 내용 |
|---|---|
| 프레임워크 | Next.js 16 (Turbopack) · Expo |
| 백엔드 | Supabase (Postgres + Auth + Storage + RLS) |
| AI | Gemini 2.5 Flash (`@google/genai`) — Q&A 답변 초안 |
| 테스트 | Vitest · Playwright (E2E) |

## 실행

```bash
pnpm install
cd apps/web && pnpm dev        # http://localhost:3000
```

- 환경변수: `apps/web/.env.local` (Supabase 키, 초대 코드, Gemini 키 등)
- DB 마이그레이션: `supabase db push` (수동 실행)
- 검증: `npx tsc --noEmit && npx vitest run`

## 계정과 권한

역할 4종 + 관리자 플래그. 로그인 방식이 역할마다 다름.

| 역할 | 로그인 | 권한 |
|---|---|---|
| 관리자 (`is_super_admin`) | 이메일 | 선생님 전체 + 타 스태프 계정 정리 |
| 선생님 `teacher` | 이메일 (초대 코드 가입) | 전체 메뉴 |
| 사무 조교 `ta_desk` | 이메일 (초대 코드 가입) | 수업 운영 메뉴 (통계·모니터링·상담 제외) |
| 첨삭 조교 `ta_assistant` | 이메일 (초대 코드 가입) | Q&A · 쪽지만 |
| 학생/학부모 | **전화번호** + 비밀번호 | 학생 대시보드 (자가 가입 없음) |

- 학생 계정은 스태프가 생성. 초기 비밀번호는 환경변수(`INITIAL_STUDENT_PASSWORD`), 첫 로그인 시 변경 강제 (`must_change_password`)
- 스태프 가입은 초대 코드로 역할 자동 결정 (`TEACHER_INVITE_CODE` / `TA_ADMIN_INVITE_CODE` / `TA_ASSISTANT_INVITE_CODE`)
- **삭제 시맨틱 주의**: 스태프 삭제 = `is_active=false` 소프트 비활성화 (FK RESTRICT 때문에 기록 보존). 학생 삭제 = 하드 삭제 (CASCADE)

## 스태프 기능 (18개 메뉴)

| 메뉴 | 요약 |
|---|---|
| 홈 | 오늘 시간표, 분반/학생/스태프 현황, 추가 일정(보강) 등록, 미확인 상담 배지 |
| 분반 관리 | 분반 CRUD. 삭제는 기본 소프트(보관), 하드 삭제 별도 |
| 학생 관리 | 계정 생성(단건/엑셀 일괄), 학부모 연결, 분반 이동(이력 보존), 비밀번호 초기화, **휴원** |
| 출석 체크 | 출석/지각/결석 + 결석 사유 → 리포트 특이사항 맨 앞에 자동 표기 |
| 과제 관리 | 주차 단위 등록, 카테고리 자유 추가, 이행률은 0~100% **20% 단위 6단계만** |
| 테스트 점수 | 테스트 등록(문항/난이도/만점), 단건·일괄 점수 입력, 평균 자동 계산 |
| 특별 시험 | 모의고사 등 별도 관리, 등급컷·학습 제안, 시험명+날짜 기준 **자동 석차** |
| 학습 리포트 | 출석·점수·과제 자동 취합 → 이미지 생성 → 카카오 발송. 분반 일괄 생성/발송 |
| 공지사항 | 전체/분반 대상, 상단 고정, 분반 공지 시 재원생 자동 쪽지+알림, 읽음 추적 |
| 강의 영상 | 강좌 단위 관리, 분반별 접근 권한 |
| 질의응답 | 상태 흐름: 대기 → 담당 지정 → 답변 완료. AI 초안(힌트/최종답 2모드), 답변 수정은 본인 것만(선생님은 전체) |
| 상담 & 문의 | 외부 상담 신청 + 재원생 1:1 문의 통합, "쪽지 보내기" 클릭 시 학생 자동 선택 |
| 쪽지 발송 | 개인/분반 발송, 시스템 자동 쪽지(`is_system`)는 내 발송함에서 제외 |
| 답변 통계 | 조교별 답변 건수·응답 속도 |
| 모니터링 | 로그인(7일, E2E 봇 제외), AI 사용량·예상 비용(원화) |
| 감사 로그 | 로그인, 계정 생성/삭제, 비밀번호 초기화 등 기록 |
| 오류 로그 | 시스템 오류 수집 |

### 휴원

- 시작일~종료일 지정 (`suspended_from` / `suspended_until`)
- 기간 중: 학생 화면 상단 배너 표시, **새 질문 등록·리포트 조회 차단** (홈·학습·공지는 접근 가능, 로그인 차단 아님)
- 종료일 경과 시 **자동 해제** (날짜 비교 방식, 별도 배치 없음). 조기 해제 버튼 있음

## 학생/학부모 화면 (웹 + 모바일 동일)

| 탭 | 요약 |
|---|---|
| 홈 | D-Day 카드, 오늘 시간표, 할 일(직접 관리), 최근 공지, 휴원 배너 |
| 학습 | 과제 목록·이행률, 출석 이력, 분반에 열린 강의 영상 |
| QnA | 사진 첨부 질문 → 조교 답변, 답변 알림 |
| 리포트 | 발행된 학습 리포트 열람 |
| 더보기 | 공지 전체, 쪽지함(전체 읽음), 1:1 문의, 비밀번호 변경 |

## 자동 동작

- Q&A 답변 등록 → 학생에게 알림 + 시스템 쪽지
- 분반 공지 등록 → 재원생 전원에게 `[공지] 제목` 쪽지 + 알림
- 휴원 종료일 경과 → 자동 해제
- 새 상담/문의 → 선생님 사이드바 미확인 배지
- 실시간 알림 벨 (Supabase Realtime), 공지 읽음 자동 기록
- 특별 시험 자동 석차, 감사 로그 자동 기록

## 아키텍처 메모

- **인증 헤더 포워딩**: `proxy.ts`(미들웨어)에서 검증한 사용자 정보를 신뢰 헤더로 전달 → `getVerifiedUser()`가 네트워크 호출 없이 읽음. 헤더는 매 요청 삭제 후 재설정(스푸핑 방지)
- **RLS**: `get_my_role()` 기반 정책. 학생은 스태프 `users` 행 조회 불가 → 답변자 이름 표시 등 특정 조회만 admin 클라이언트로 우회
- **Next.js 16 주의**: `apps/web/AGENTS.md` 참고 — 학습된 Next.js와 API가 다름. `node_modules/next/dist/docs/` 먼저 확인
- **마이그레이션 append-only**: 기존 파일 수정 금지, CI(`migration-guard.yml`)가 차단

## 알려진 예외 케이스

- 스태프 삭제 후에도 그 사람의 답변·공지·쪽지는 남음 (의도된 동작)
- 휴원생은 로그인 가능 — 휴원은 기능 제한이지 계정 차단이 아님
- AI 초안은 조교 검토 후 발송 — 학생에게 자동 전송되지 않음
- 자동 알림 쪽지는 실제 답변자 이름으로 발송되나 발송자의 "내 발송함"에는 미표시
- 학생 일괄 등록은 행 단위 부분 성공 — 실패 행(주로 전화번호 중복)만 사유와 함께 반환
- 분반 이동 시 이전 소속은 `is_active=false` 이력으로 보존
